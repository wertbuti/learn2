سه نوع وب سرویس وجود دارد
1- soap
2- rest
3-XML-RPC
---------------------------------------
مشخصات soap
1- روش قدیمی تر است
2- خروجی به صورت فایل xml است
3- سرعت ان کم میباشد و پهنای زیادی از باند را اشغال میکند
4- دارای سه قانون اساسی با نامهای soap و WSDL(ویزدل) و UDDI میباشد
5- قانون اول یعنیی soap بیان میکند که هر درخواست یا پاسخ باید در تگ Envelop باشد و داخل آن دو تگ هد و بادی قرار داشته باشد
6- قانون دوم یعنی WSDL در واقع کاتالوگ راهنمای استفاده از وب سرویس میباشد و این فایل به فرمت xml است
7- قانون سوم یعنی UDDI هم در واقع محل قرار گیری تمام WSDL ها را بیان میکند یعنی آدرس ذخیره شدن کاتولوگ های راهنما را UDDI میگویند
---------------------------------------
مشخصات Rest
1- روش جدیدتریس است
2- خروجی انواع مختلفی از جمله xml و json و یا حتی متن ساده دارد
3- سرعت آن زیاد است و پهنای کی از باند را اشغال میکند
4- مستقل از وضعیت است
5- قابل ذخیره سازی در کلاینت است(شماره نسخه دارد)
6- ...
-----------------------------
api
در واقع ارتباط دو اپلیکشن با هم از طریق api صورت میگیرد
اگر این ارتباط در بستر وب باشد به آن web api میگویند
و دو نوع از ارتباط api در بستر وب به نامهای soap api (همان وب سرویس soap ) و rest api (همان وب سرویس rest) میباشد
